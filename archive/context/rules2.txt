# 0. **Universe**

### **Rule U1 ‚Äî The whole universe is one file: `melvin.m`**

It contains:

* node table
* edge table
* raw bytes (data + machine code)
* small control buffers

Nothing exists outside the file except the CPU running it.

### **Rule U2 ‚Äî The runner does nothing except:**

* map `melvin.m` into memory
* jump into bytes when asked
* expose syscalls
* sync the file

No logic. No interpretation. No policies.

# 1. **Objects**

## Nodes

### **Rule N1 ‚Äî A node is a slot that can hold energy.**

Each node has:

* an ID
* a small energy value
* a little memory (fatigue, bias, trace)
* a list of outgoing edge indices

Nodes have **no type**.
A node can mean anything depending on structure.

## Edges

### **Rule E1 ‚Äî An edge is a directed connection between two nodes.**

It has:

* source node
* destination node
* weight
* trace

Edges have **no type**.
They only carry energy.

## Raw Bytes

### **Rule B1 ‚Äî Any byte region in `melvin.m` can be data or machine code.**

There is no difference.
The graph decides what bytes mean.

# 2. **Energy**

## Pulses

### **Rule P1 ‚Äî Energy flows in pulses.**

A pulse is:
‚Äúnode i receives energy e‚Äù.

### **Rule P2 ‚Äî External sensors inject pulses.**

Cameras, mics, files, motors, etc. all produce energy pulses when read.

### **Rule P3 ‚Äî Internal loops generate pulses.**

Edges send energy around the graph.

### **Rule P4 ‚Äî Background noise occasionally creates tiny pulses.**

This prevents total silence.

---

# 3. **Node Dynamics**

### **Rule ND1 ‚Äî Nodes accumulate energy.**

When a node receives a pulse, it adds it to its stored energy.

### **Rule ND2 ‚Äî Nodes fire when energy passes a threshold.**

When firing:

* take a fraction of stored energy
* push it out along outgoing edges
* keep the rest
* gain fatigue

### **Rule ND3 ‚Äî Nodes leak energy every tick.**

Stored energy decays slightly each tick.

### **Rule ND4 ‚Äî Nodes become temporarily harder to fire if they fire too much.**

Fatigue increases threshold.
Fatigue decays slowly over time.

---

# 4. **Edge Dynamics**

### **Rule ED1 ‚Äî Energy flow over an edge is proportional to its weight.**

More weight ‚Üí easier to pass energy.

### **Rule ED2 ‚Äî Edges that carry energy strengthen.**

Edge weight increases when pulses flow along it.

### **Rule ED3 ‚Äî Edges that don‚Äôt get used decay.**

Slow decay every tick.

### **Rule ED4 ‚Äî New edges can appear between co-active nodes.**

If two nodes fire in the same tick, a new edge may form.

### **Rule ED5 ‚Äî Bad edges disappear over time.**

Edges with no use go to zero and are removed.

# 5. **Structure Formation**

### **Rule S1 ‚Äî Stable loops of nodes become molecules.**

If a set of nodes repeatedly send energy in a loop, they become a stable circuit.

### **Rule S2 ‚Äî Molecules combine into larger motifs.**

Two circuits that often activate together link and form a bigger structure.

### **Rule S3 ‚Äî Motifs form modules.**

Collections of motifs that frequently co-activate become a subsystem.

Structure is emergent, not defined.

# 6. **Execution (how graph influences CPU/OS)**

### **Rule X1 ‚Äî A small execution buffer exists in the file.**

Graph can write:

* `code_offset`
* `entry_point_offset`
  inside this buffer.

### **Rule X2 ‚Äî If execution buffer is set, runner jumps into that code.**

CPU executes bytes directly out of `melvin.m`.

### **Rule X3 ‚Äî Code running in `melvin.m` may:**

* perform syscalls (read camera, write motor, etc.)
* read/write any byte in `melvin.m`
* modify its own code region
* modify nodes and edges

### **Rule X4 ‚Äî The graph learns which code offsets cause useful energy.**

There is no special ‚ÄúMC function‚Äù.
There are only **bytes**.

The graph decides which ones to run.

# 7. **Selection (Why circuits survive)**

### **Rule SEL1 ‚Äî Nodes track energy spent and energy returned.**

After firing, a node watches how much energy arrives later.

### **Rule SEL2 ‚Äî Return-on-energy shapes learning.**

If firing tends to bring in more energy:

* outgoing edges strengthen more
  If it brings less:
* outgoing edges strengthen less
* or weaken

### **Rule SEL3 ‚Äî Circuits that cause more future energy dominate.**

Useful behaviors survive (motor moves, sensor reads, file loads).
Useless circuits die.

No goals.
No reward.
Just selection pressure.

Nodes don‚Äôt mean anything alone.

### **Rule M2 ‚Äî Thinking is energy flow through structure.**

If a motif tends to put ‚Äúrun code at offset X‚Äù into the buffer, that code runs.
